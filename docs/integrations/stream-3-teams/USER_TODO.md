# Stream 3 — Microsoft Teams Compliance Recording: User-only checklist

This is the human-only side of Stream 3. The Python scaffold landed in
the same PR as this file; everything below is something Claude cannot
do for you. See ``CERTIFICATION_PATH.md`` for the full background and
architecture rationale.

The order matters — items 1 and 2 unblock everything else.

## Microsoft tenancy & partnership

- [ ] **Sign up as a Microsoft Partner** at
      https://partner.microsoft.com/. Required to receive a verified
      publisher consent prompt and to submit for compliance certification.
      Lead time: 4–8 weeks for verification.
- [ ] **Provision a dedicated Azure tenant + subscription** that will
      host LINDA's media bot. This is *your* tenant, not a customer's.
- [ ] **Decide on hosting**: Azure App Service for Containers (start
      here) vs Azure Container Apps (recommended for production scale)
      vs AKS. Document the choice; it affects the .NET bot's deployment
      pipeline and certification submission.

## Azure AD app registration (the bot identity)

- [ ] **Register a multi-tenant AAD app** in your Azure tenant.
- [ ] **Add application (not delegated) permissions**:
  - [ ] ``Calls.AccessMedia.All``
  - [ ] ``Calls.JoinGroupCallAsGuest.All``
  - [ ] ``OnlineMeetingArtifact.Read.All``
- [ ] **Grant tenant admin consent** in your tenant.
- [ ] **Create a client secret** (or upload a certificate — preferred
      for production) and store it in your secrets manager.
- [ ] **Set the three env vars** on the LINDA backend deployment:
  - [ ] ``TEAMS_BOT_APP_ID`` (the AAD app's client id)
  - [ ] ``TEAMS_BOT_APP_SECRET`` (the secret, NOT a delegated user
        token; this is the app-only credential)
  - [ ] ``TEAMS_TENANT_ID`` (your AAD tenant id, not the customer's)
- [ ] **Provide a public HTTPS notification URL** (e.g.
      ``https://api.linda.example.com/api/v1/teams/notification``) and
      verify it's reachable from Microsoft's IP ranges.

## Decision: build vs license the .NET media bot

- [ ] **Choose**: build the .NET media bot in-house, or license a
      certified bot (Verint, ASC, Theta Lake, Numonix, …) and integrate
      it via the ``MediaBot`` interface.
      - Build: 3–6 engineer-months, full control, certification owned
        by us.
      - License: faster to market, ongoing per-seat fees, certification
        owned by the partner.
- [ ] If building, **fork the Microsoft sample** at
      https://github.com/microsoftgraph/microsoft-graph-comms-samples
      and budget for the .NET workstream. **This is out of scope for
      Stream 3 — flag it to whoever picks up the follow-on.**
- [ ] If licensing, **wire a ``MediaBot`` adapter** that calls the
      partner's API, replacing the stub at
      ``backend/app/services/teams_recording/bot_interface.py``.

## Microsoft compliance recording certification

- [ ] **Open a Microsoft Partner Center support case** under the
      "Teams compliance recording" certification track.
- [ ] **Prepare the certification kit**:
  - [ ] End-to-end demo video of a Teams call being recorded.
  - [ ] Architecture diagram with explicit data-flow boundaries.
  - [ ] Security review (pen-test report, secret rotation policy,
        encryption-at-rest evidence).
  - [ ] Privacy posture (DPA, retention defaults, data-residency map).
- [ ] **Budget 3–9 months** for the back-and-forth with Microsoft's
      certification engineer.
- [ ] **Plan for a re-submission** — first-pass failures are common.

## Customer-side rollout (per customer, after bot is certified)

For every customer that wants Teams compliance recording:

- [ ] **Confirm the customer has Microsoft 365 E5 or A5** licensing.
      Compliance recording is not available on lower SKUs.
- [ ] **Send the customer's Teams admin the PowerShell template** —
      generated by ``backend.app.services.teams_recording.policy.render_powershell``
      with our ``TEAMS_BOT_APP_ID``. The script:
  1. Registers our bot as a compliance recording application.
  2. Creates the ``LINDA-CompliancePolicy`` policy.
  3. Attaches the bot to the policy.
- [ ] **The customer admin must run the script** on a workstation with
      the MicrosoftTeams PowerShell module installed and a Teams admin
      account.
- [ ] **The customer admin grants the policy to specific users**:
      ``Grant-CsTeamsComplianceRecordingPolicy -Identity user@... -PolicyName LINDA-CompliancePolicy``.
- [ ] **Provide LINDA's notification URL** to the customer's
      networking team for any egress allowlist they enforce.

## Storage & data residency (when the bot lands)

- [ ] **Decide on audio storage**: Azure Blob Storage in your tenant
      vs the customer's tenant vs LINDA's existing S3.
- [ ] **Set retention defaults** — Microsoft does not retain bot media
      for you.
- [ ] **Document data-residency claims** (US / EU / per-tenant
      isolation) and align with the existing tenant data-residency
      story in this repo.
